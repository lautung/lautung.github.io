# 启动优化

启动体验决定用户对应用的第一印象。启动优化要先测量，再定位耗时点。

## 关注指标

- 冷启动：进程不存在时启动。
- 温启动：进程存在但 Activity 需要重建。
- 热启动：应用从后台快速回到前台。

## 优化方向

- 减少 Application 初始化工作。
- 延迟非必要 SDK 初始化。
- 避免主线程 IO。
- 控制启动页和首页首帧时间。

## 学习建议

先用 Android Studio Profiler 和系统日志观察启动时间，再逐项拆分初始化任务。
